# まとめ

- コメントはメンテされにくいため注意すること
- コメントにただ実装を辿っただけの情報を書かないようにする
- そもそもコメントがいらないシンプルな設計を心がける
- コメントは「変更する時」に注意してほしいことを書く
- 「コメントが悪」と上辺だけの理解をしないように
  メソッド名に長々と説明を書いたら同じこと
- コメントを書くにしても言語仕様やスタンダードな周辺ツールに合わせたコメントの書き方だと良いね
